# Onboarding Documentation (Enterprise) - Engineering

## Objective
Enable new engineer to contribute safely within 5 working days.

## Access Checklist
- Source repository
- CI/CD dashboard
- Observability stack (logs, metrics, tracing)
- Issue tracker and wiki
- VPN and production read-only access (if applicable)

## 5-Day Plan
### Day 1
- Security and compliance briefing
- Read architecture and product context
### Day 2
- Local setup and run core services
### Day 3
- Debug walkthrough using real incident replay
### Day 4
- Shadow code review + test execution
### Day 5
- Deliver first low-risk ticket to staging

## Mandatory Learning Path
- BRD/PRD/SRS baseline
- HLD/LLD/TSD
- Runbook and incident process
- Git workflow and review standards

## Readiness Criteria
- Can trace request end-to-end with traceId
- Can open PR with correct template and checks
- Can execute rollback on staging with mentor
