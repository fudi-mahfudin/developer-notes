# Documentation Index

Daftar dokumen yang paling sering digunakan dalam software development.

## Requirement & Product

- **BRD (Business Requirements Document)** - tujuan bisnis, ruang lingkup, KPI, stakeholder.
- **PRD / FRD / FSD (Product/Functional Requirements/Functional Specification)** - kebutuhan fungsional, user flow, acceptance criteria.
- **SRS (Software Requirements Specification)** - spesifikasi kebutuhan sistem secara formal.

## Technical & Architecture

- **TSD (Technical Specification Document)** - solusi teknis, desain komponen, dependency.
- **HLD (High-Level Design)** - gambaran arsitektur level atas.
- **LLD (Low-Level Design)** - detail implementasi level modul/service.
- **API Documentation (OpenAPI/Swagger/Postman)** - endpoint, payload, auth, error handling.
- **Database Documentation (ERD/Data Dictionary)** - model data, relasi, definisi tabel/kolom.

## Delivery & Project Tracking

- **JIRA / Issue Tracker** - backlog, user story, task, bug, sprint progress.
- **Roadmap & Milestone Plan** - rencana delivery per fase.
- **Release Notes** - perubahan per rilis, dampak, known issues.
- **Change Log** - histori perubahan versi aplikasi/paket.
- **BAST (Berita Acara Serah Terima)** - dokumen serah terima pekerjaan/deliverable.

## Quality & Testing

- **Test Plan** - strategi, scope, environment, risiko pengujian.
- **Test Case / Test Scenario** - langkah uji, expected result, status.
- **RTM (Requirement Traceability Matrix)** - pelacakan requirement ke test.
- **UAT Documentation** - skenario dan hasil user acceptance test.
- **Defect/Bug Report** - detail reproduksi, severity, root cause.

## Engineering Workflow

- **Git Workflow Documentation** - branching strategy, commit convention, PR flow.
- **Code Review Checklist** - standar review kualitas, keamanan, performa.
- **CI/CD Documentation** - pipeline build-test-deploy, rollback strategy.
- **Runbook / SOP Operasional** - prosedur incident, maintenance, recovery.

## Knowledge Management

- **Confluence / Wiki** - dokumentasi kolaboratif lintas tim.
- **ADR (Architecture Decision Record)** - keputusan arsitektur beserta alasan.
- **Onboarding Documentation** - setup project, struktur repo, cara kerja tim.
- **FAQ / Troubleshooting Guide** - isu umum dan langkah penyelesaian.
